RMI 注册表（RMI Registry）是 Java RMI（Remote Method Invocation，远程方法调用）框架中的一个重要组件。它的作用是一个目录服务，专门用于存储并管理远程对象的引用，使得客户端能够通过名字查找到并访问这些远程对象。

RMI 注册表的功能

  1. 管理远程对象引用：

    • RMI 注册表维护了一个远程对象引用表，每个对象都通过唯一的名字（通常是字符串）绑定到注册表中。客户端可以通过这个名字查找和获取远程对象的引用。

  2. 支持分布式应用中的远程调用：

    • 在分布式应用中，客户端和服务端位于不同的 JVM（Java 虚拟机）上，通常在不同的物理机器上。RMI 注册表帮助客户端查找到服务端提供的远程对象，使得客户端能够调用远程方法。

  3. 作为远程对象的目录：

    • RMI 注册表像一个目录，客户端可以查找（lookup）它所需要的远程服务。每个远程对象都需要通过一个名字绑定（bind）到注册表中，这样客户端才能通过该名字查找到并使用它。

RMI 注册表的使用

通常，RMI 注册表在服务端启动，并通过 rmiregistry 命令来运行。服务端程序会将远程对象绑定到注册表中，使得客户端能够通过名字来获取对象引用。

总结

  • RMI 注册表的作用是提供一个目录服务，管理远程对象引用，使客户端能够通过名字查找到远程对象。
  • 注册表的工作原理是在服务端将对象绑定到注册表，客户端通过名字查找，获取对象的引用。
  • RMI 注册表通常在默认端口 1099 上运行，但也可以配置其他端口。

通过 RMI 注册表，Java RMI 实现了分布式系统中远程对象的管理和访问。
