接口和父类（即基类或超类）都是 Java 中实现代码复用和结构化设计的手段，但它们有着不同的设计目的和使用场景。以下是接口和父类的关键区别：

1. 设计意图

  • 父类（基类）：用于表示“is-a”（是一个）关系，通常是具体的事物的抽象。父类通常包含通用的属性和行为，子类继承父类以复用这些通用代码。例如，一个 Animal 父类可以定义 eat() 方法，而子类 Dog、Cat 可以继承这个方法并进行扩展。
  • 接口：用于表示“can-do”（可以做）或“能力”的关系，是行为的约定。接口定义了一组方法，而具体的实现类通过实现接口来提供这些方法的具体实现。例如，Runnable 接口代表“可以运行”的能力，而不表示具体事物的类型。

2. 多重继承

  • 父类：Java 不支持多重继承，即一个类只能有一个直接父类。这是为了避免“钻石继承”问题（多个父类中同名方法冲突的情况）。
  • 接口：Java 支持多重接口继承。一个类可以实现多个接口，从而灵活地组合不同的行为。例如，一个类可以同时实现 Runnable 和 Serializable 接口，既支持多线程运行，也支持序列化。

3. 实现方式

  • 父类：父类可以包含具体实现（即方法体）和成员变量。子类继承这些成员，可以选择使用、重写（override）父类方法，或在子类中添加新的功能。
  • 接口：接口在 Java 8 之前只允许包含方法签名（不包含实现）和静态常量。Java 8 引入了默认方法和静态方法，Java 9 又引入了私有方法，使接口可以有默认实现，但接口仍不能包含实例变量。

4. 抽象与实现的关系

  • 父类：父类主要用于代码复用和结构组织。父类是一个实现的基础，子类继承父类的代码，并在必要时进行扩展。
  • 接口：接口是行为的契约，通常不包含实现。实现类承诺遵守接口规定的行为契约，而接口则定义了哪些方法必须被实现类提供。

5. 约束与扩展性

  • 父类：父类提供的实现是一种更强的约束，子类无法摆脱父类中的某些实现（除非重写）。这在某些场景下可能限制了扩展的灵活性。
  • 接口：接口提供的是方法签名的约束，而具体的实现则由实现类来提供，这使得实现类有很大的自由度。

使用场景示例

  • 父类：当多个类有相同的属性和行为且共享实现时，用父类。例如，一个 Vehicle 类有 move() 方法，Car 和 Bike 继承 Vehicle 并共享 move() 的实现。
  • 接口：当多个类需要具备相同的行为特性，但这些行为的具体实现因类而异时，用接口。例如，一个 Flyable 接口表示“可以飞”的能力，Bird 和 Airplane 都可以实现 Flyable，但它们的飞行方式不同。

总结

  • 父类：用于描述事物的抽象模型，带有具体实现，限制了多重继承。
  • 接口：用于描述行为的契约，支持多重实现，提供较大的灵活性。
